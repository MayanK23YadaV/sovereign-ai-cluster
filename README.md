## 📖 Overview
The Sovereign AI Cluster is an enterprise-grade automation stack that transforms standard desktop PCs into a unified AI supercomputer. By abandoning traditional, brittle "USB cloning" methods in favor of dynamic network discovery and **Ansible Orchestration**, this repository enables the instant deployment, configuration, and scaling of a 30+ node computing cluster.

### 🌟 Proven Features
Every script and playbook in this repository has been field-tested and proven to work on production hardware:

* **🕸️ Dynamic Network Discovery:** Automatically scans the `172.16.x.x` subnet to detect fresh Linux Mint installations, injects SSH keys securely using `sshpass`, and dynamically builds an Ansible inventory.
* **🚀 Distributed Inference:** Seamlessly slices and distributes massive GGUF models (like DeepSeek-R1 or Llama-3) across the network using the `llama.cpp` RPC engine.
* **⚡ Bare-Metal Performance Tuning:** Automatically locks CPU scaling governors to `performance` mode and masks system-level `sleep`, `hibernate`, and GUI screen blanking to ensure maximum compute availability.
* **🎨 Visual Identity:** Automatically provisions unique Hostnames, custom ASCII MOTD banners, and role-based shell prompt coloring (🔴 Red for Head Node, 🟢 Green for Workers) to prevent operator error.
* **🪟 Automated Dual-Boot Prioritization:** Dynamically searches the host's GRUB configuration and sets the `Windows Boot Manager` as the #1 priority, ensuring nodes reboot to Windows by default unless explicitly told to join the Swarm.
* **🌐 Secure Remote UI:** Deploys a permanent `llama-server` background service on the Head Node, allowing users to securely port-forward a ChatGPT-style Web UI over a Cloudflare Tunnel (`ssh -L 8080:localhost:8080`).

## 🚀 Getting Started

### Prerequisites
1. A **Head Node** running Linux Mint.
2. Multiple **Worker Nodes** fresh-installed with Linux Mint and connected to the same local subnet.
3. Ansible and `sshpass` installed on the Head Node.

### Step 1: Discover & Assimilate Nodes
Power on the new worker nodes. On the Head Node, run the discovery script. It will scan the network, ask for the default credentials you set during the Linux Mint install, and automatically inject your SSH keys.
```bash
sudo bash scripts/discover_nodes.sh
```

### Step 2: Deploy the Swarm Architecture
Once the `hosts.ini` is generated by the discovery script, run the master Ansible playbook. This will rename the nodes, tune their CPUs, disable sleep timers, set up dual-boot priorities, and deploy the `llama-rpc` engine.
```bash
ansible-playbook -i /etc/ansible/hosts.ini ansible/cluster_init.yml --extra-vars "ansible_become_pass=root"
```

### Step 3: Monitor the Cluster
Check the live health, RAM usage, and RPC status of all assimilated nodes:
```bash
bash scripts/status.sh
```

### Step 4: Run Distributed AI
Start the `llama-server` on the Head Node, pointing it to your model and the IPs of your newly assimilated RPC workers:
```bash
/opt/llama.cpp/build/bin/llama-server -m /mnt/ramdisk/model.gguf --host 0.0.0.0 --port 8080 -ngl 99 --rpc 172.16.x.x:50052,172.16.x.y:50052 -c 4096
```

## 🔒 Security
* **No Hardcoded Passwords:** The discovery script requests credentials strictly via ephemeral standard input.
* **Root Lockdown:** The Ansible playbook automatically disables password-based `root` SSH access on all worker nodes after initial key injection.
* **Cloudflare Zero-Trust:** External SSH access to the Head Node is handled strictly via Cloudflare Tunnels; port 22 is never exposed to the public internet.
